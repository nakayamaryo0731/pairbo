# Pairbo マーケティング戦略

## 概要

Pairbo（ペアボ）のマーケティング戦略。
個人開発のため **コンテンツマーケティング + SEO** を中心に、計測基盤を整えたうえで効果を見ながら施策を拡大する。

**現状**: ソフトローンチ済み（公開中、未告知）

## ターゲット層

### プライマリターゲット

**お財布別のカップル・夫婦**

- 同棲開始〜数年目のカップル（20代中心）
- 共働き夫婦でお財布は別管理（30代中心）
- 「割り勘」「傾斜折半」にニーズがある層

### ペルソナ例

```
名前: 田中さん（28歳）& 鈴木さん（26歳）
状況: 同棲開始3ヶ月目、お財布は別々
課題:
- 収入差があるので50:50は不公平感がある
- スプレッドシートは面倒で続かない
- 相手にアプリをインストールさせるのはハードルが高い
- 共同口座やカードを作るのも面倒
```

### Pairbo の強み

1. **Web版（PWA対応）**: アプリインストール不要、ホーム画面追加でネイティブ感覚
2. **URL招待**: 相手にURLを送るだけで即参加。「アプリ入れて」のハードルがない
3. **傾斜折半**: 収入差があるカップルに最適。割合指定・金額指定・全額負担に対応
4. **常に最新版**: アップデート不要、ストレージを圧迫しない

### 競合との棲み分け

- **vs Shareroo**: プラットフォーム非依存 + URL招待が差別化。Shareroo はネイティブアプリ限定
- **vs Splitwise**: 傾斜折半対応 + 日本語ネイティブ。Splitwise は均等割りがメインで日本語UIが弱い
- **vs OsidOri**: お財布別のまま使える。OsidOri は共同口座・資産管理寄りで、ライト層には重い
- **vs スプレッドシート**: 精算自動計算 + リアルタイム同期。スプレッドシートは続かない

## 基本戦略: 「計測 → コンテンツ → 判断」

### 順序

1. **計測基盤を整える**（GA イベント、Search Console）→ 効果測定を可能に
2. **無料施策を実行**（Zenn記事、X投稿、プロダクト紹介サイト）
3. **データを見て有料施策を判断**

### 発信方針: 個人アカウントで発信

- 個人開発者の「作ってる人」感が信頼に繋がる
- フォロワーゼロのプロダクトアカウントよりリーチが出る
- 開発過程の発信が個人開発者コミュニティに刺さる
- プロダクトアカウントは必要になってから作る

### Phase 3: 効果測定 → 判断

Phase 1-2 を1-2ヶ月実施後、Search Console と GA のデータを見て判断:

- **検索流入が増えている** → SEO 記事を増やす
- **X 経由の登録がある** → X 投稿頻度を上げる / プロダクトアカウント検討
- **Zenn 記事経由が多い** → 技術記事を量産
- **どれも微妙** → 有料広告（X ads, Google ads）を少額テスト

### やらないこと（今は）

- プロダクト専用 SNS アカウント作成（個人アカウントで十分）
- 有料広告（計測基盤なしに出しても効果判断不能）
- Instagram（画像制作コストが高い。X の方がテキストベースで個人開発者と相性が良い）
- Product Hunt（英語 LP ができてから）
- インフルエンサーマーケ（規模感が合わない）

## KPI

GA イベントベースで計測する。

### 認知（Search Console + GA）

- 検索インプレッション数
- 検索クリック数・CTR
- LP ページビュー数
- 流入元別セッション数

### 獲得（GA カスタムイベント）

- `sign_up`: 新規ユーザー登録数
- `create_group`: グループ作成数
- `invite_sent`: 招待リンク発行数

### アクティベーション（GA カスタムイベント）

- `create_expense`: 支出登録数
- 初回支出登録までの時間

### 収益化（GA カスタムイベント）

- `upgrade_premium`: プレミアムアップグレード数
- Free → Premium 転換率

### リテンション

- WAU（週間アクティブユーザー）
- 支出登録数/ユーザー
- 精算完了率
